An Introduction

Let's see what's so special about JavaScript what we can achieve with it, and what other technologies play well with it.

What is JavaScript

JavaScript was initially created to "make web pages alive".

The programs in this language are called scripts. They can be written right in a web page's HTML and run automatically as the page loads.

Scripts are provided and executed as plain text. They don't need special preparations or compilation to run.

In this aspect, JavaScript us very different fram another language called Java.

When JavaSript was created, it initially had another name: "LiveScript". But Java was very popular at that time, so it was decided that positioning a new language as a "younger brodther" if Java would help.

But as it evolved, JavaSript became a fully independedt languange with its own specifications called ECMAScript, and now it has no relation to Java at all.

Today, javaScript can execute not only in the browser, but also on the server, or actually on any devices that has a special program called the JavaSript engine.

How do engines work?

Engines are complicated. but the basics are easy.

  • 1. The engine (embedded if it's a browser) reads ("parses") the script.

  • 2. Then it converts ("compiles") the script to the machine language.

  • 3. And then the machine code runs, pretty fast.

The engine applies optimization at each step of the process. It even watches the compiled scripts as it runds, analyzes the data that flows through it, and further optimizes the machine code based on that knowledge.
